## 2. 基础课程适配分析

### ✅ 适合基础课程的实验（4 个）

#### 实验一：客户支持旅程映射

**原版内容**：
- 使用 Whiteboard 设计理想客户支持旅程
- 生成 18 个支持流程步骤
- 自动分类整理
- 识别痛点

**适配方案**：
```
实验名称：客户支持旅程映射
输入：支持流程主题（如"产品退换货流程"）
输出：support_journey.md

核心技能：
- 生成支持流程步骤
- 分类整理（按阶段）
- 识别客户痛点
- 生成总结文档

提示词示例：
"请为产品退换货流程生成支持旅程映射。

包含：
1. 支持流程步骤（12-15 个）
2. 按阶段分类（受理/处理/解决/跟进）
3. 客户痛点识别（8-10 个）
4. 流程总结

保存为：support_journey.md"
```

**技术难度**：⭐⭐（文本生成 + 分类）  
**工作量**：4-6 小时

## 3. 高阶课程适配分析（Agent 方向）

### ✅ 适合高阶课程的实验（2 个）

#### Agent 一：Support Analyst Agent（基于 Task 4）

**原版内容**：
- 分析 90 天支持案例数据
- 识别问题模式和趋势
- 生成可视化仪表板
- 创建演示幻灯片

**Agent 设计**：
```yaml
# agent_support_analyst.yaml
name: Support Analyst Agent
description: 客服数据分析专家
version: 1.0

capabilities:
  - case_analysis: 案例数据分析
  - pattern_detection: 模式识别
  - trend_analysis: 趋势分析
  - visualization: 可视化生成
  - dashboard_creation: 仪表板创建

presets:
  - name: case_analysis
    description: 案例数据分析
    prompt: |
      请分析以下支持案例数据：
      - 数据文件：{data_file}
      - 时间范围：{time_range}
      
      分析维度：
      1. 问题类别分布（Top 5）
      2. 严重程度混合
      3. 受影响模块
      4. 升级率
      5. 解决时间分布
      
      输出：简明简报 + 关键洞察。
      
  - name: correlation_analysis
    description: 相关性分析
    prompt: |
      请测试以下相关性：
      1. 严重程度 vs 升级率
      2. 模块 vs 平均解决时间
      3. 客户类型 vs 满意度
      
      输出：
      - 相关系数
      - 可视化图表
      - 关键洞察
      
  - name: dashboard_creation
    description: 仪表板创建
    prompt: |
      请创建支持数据仪表板：
      
      包含：
      1. 关键指标卡片
      2. 趋势图表
      3. 问题分布图
      4. 关键洞察标注
      
      输出：单页仪表板（PPT 格式）

output_formats:
  - markdown: 分析简报
  - xlsx: 数据分析表
  - pptx: 演示幻灯片
```

**教学价值**：
- ✅ 客服数据分析方法论
- ✅ 模式识别和趋势分析
- ✅ 数据可视化
- ✅ 仪表板设计

**工作量**：16-20 小时

#### Agent 二：Troubleshooting Guide Agent（基于 Task 7-10）

**原版内容**：
- 创建故障排查指南
- 生成步骤化流程
- 包含诊断测试和升级规则
- 可视化流程图

**Agent 设计**：
```yaml
# agent_troubleshooting_guide.yaml
name: Troubleshooting Guide Agent
description: 故障排查指南专家
version: 1.0

capabilities:
  - guide_structure: 指南结构设计
  - triage_flow: 分诊流程
  - diagnostic_tests: 诊断测试
  - escalation_rules: 升级规则
  - flowchart_generation: 流程图生成

presets:
  - name: guide_structure
    description: 指南结构设计
    prompt: |
      请为以下产品线创建故障排查指南：
      - 产品线：{product_line}
      - 受众：{audience}
      
      输出结构：
      1. 分诊流程（步骤化）
      2. 快速诊断测试
      3. 证据清单
      4. 常见解决方案
      5. 升级规则
      
  - name: triage_flow
    description: 分诊流程
    prompt: |
      请创建分诊流程：
      
      包含：
      1. 问题分类（性能/质量/安全/可用性）
      2. 决策点（if/then 逻辑）
      3. 分支路径
      
      输出：结构化流程 + 决策树。
      
  - name: diagnostic_tests
    description: 诊断测试
    prompt: |
      请为以下类别创建快速诊断测试：
      - 性能问题
      - 质量问题
      - 安全问题
      - 可用性问题
      
      每个类别 3 个测试，清晰简短。
      
  - name: escalation_rules
    description: 升级规则
    prompt: |
      请定义升级触发器和规则：
      
      包含：
      1. 安全升级
      2. 重复失败
      3. 出厂缺陷
      4. 保修升级
      
      输出：清晰的升级流程。

output_formats:
  - markdown: 故障排查指南
  - docx: 可分享文档
  - md: 流程图（Mermaid 格式）
```

**教学价值**：
- ✅ 故障排查方法论
- ✅ 分诊流程设计
- ✅ 诊断测试设计
- ✅ 升级规则定义

**工作量**：16-20 小时
