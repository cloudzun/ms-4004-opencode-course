## 2. 基础课程适配分析

### ✅ 适合基础课程的实验（5 个）

#### 实验一：Word 沟通简报

**原版内容**：
- 基于产品概述创建战略沟通简报
- 为高管团队准备简报内容
- 多轮迭代优化（格式/语气/长度）
- 审查缺失内容

**适配方案**：
```
实验名称：Word 战略沟通简报
输入文件：product_overview.docx
输出文件：communication_brief.docx

核心技能：
- 读取产品文档
- 生成战略简报
- 多轮迭代优化（格式/语气/长度）
- 审查和完善

提示词示例：
"请读取 materials/product_overview.docx，
生成一份战略沟通简报。

包含：
1. 产品/项目介绍
2. 关键特性和优势
3. 与公司战略目标对齐
4. 预期业务影响

保存为：communication_brief.docx

格式要求：
- 标题字体：微软雅黑，16pt，加粗
- 正文字体：微软雅黑，10.5pt
- 英文字体：Arial"
```

**技术难度**：⭐⭐（python-docx）  
**工作量**：4-6 小时

#### 实验二：Excel 沟通指标分析

**原版内容**：
- 分析邮件通讯参与度数据

- 提供改进建议

**适配方案**：

```
输入文件：newsletter_metrics.xlsx
输出文件：communication_analysis.xlsx

- 读取沟通指标数据
- 计算打开率、点击率
- 识别高/低参与度群体
- 生成可视化图表
- 提供改进建议

提示词示例：
"请读取 materials/newsletter_metrics.xlsx，
分析沟通指标数据。

要求：
1. 计算平均打开率、平均点击率
2. 识别参与度最高/最低的 3 个部门
3. 生成对比图表
4. 提供改进建议

保存为：communication_analysis.xlsx
```

**技术难度**：⭐⭐（pandas 分析）  
**工作量**：4-6 小时

## 3. 高阶课程适配分析（Agent 方向）

### ✅ 适合高阶课程的实验（2 个）

#### Agent 一：Ideas Coach 创意教练（基于 Task 7-10）

**原版内容**：
- 创意生成和活动规划
- 多渠道内容策略
- A/B 测试设计
- 视觉概念和情绪板
- 参与度指标设计

**Agent 设计**：
```yaml
# agent_ideas_coach.yaml
name: Ideas Coach Assistant
description: 创意生成和活动规划专家
version: 1.0

capabilities:
  - campaign_ideation: 活动创意生成
  - channel_planning: 渠道规划
  - ab_testing: A/B 测试设计
  - visual_concept: 视觉概念设计
  - metrics_design: 指标设计

presets:
  - name: campaign_ideas
    description: 活动创意生成
    prompt: |
      请为以下项目生成创意活动方案：
      - 项目名称：{project_name}
      - 目标受众：{target_audience}
      - 核心信息：{key_message}
      
      输出：
      1. 4 个创意活动概念
      2. 每个概念的关键承诺、语气、注意事项
      3. 渠道建议
      
  - name: channel_plan
    description: 渠道规划
    prompt: |
      请为活动创建两周的执行计划：
      
      包含：
      - 每天的渠道（Teams/SharePoint/邮件等）
      - 内容类型（帖子/文章/图片等）
      - 负责人
      
      输出：日历格式表格
      
  - name: ab_testing
    description: A/B 测试设计
    prompt: |
      请为以下渠道设计 A/B 测试：
      - 渠道：{channel}
      - 目标：{goal}
      
      输出：
      1. 版本 A 和版本 B 的文案
      2. 测试变量（语气/措辞/呼吁行动）
      3. 成功指标

output_formats:
  - markdown: 创意方案
  - xlsx: 渠道计划表
  - docx: A/B 测试文案
```

**教学价值**：
- ✅ 创意生成方法论
- ✅ 多渠道规划
- ✅ 实验设计思维
- ✅ 数据驱动优化

**工作量**：16-20 小时

---

#### Agent 二：FAQ Assistant 问答助手（基于 Task 11）

**原版内容**：
- 创建 FAQ 问答 Agent
- 配置知识库（3 个文档）
- 定义回答边界和升级规则
- 语气和一致性控制

**Agent 设计**：
```yaml
# agent_faq_assistant.yaml
name: FAQ Assistant
description: 项目问答专家
version: 1.0

capabilities:
  - project_qa: 项目问答
  - document_search: 文档检索
  - boundary_aware: 边界感知
  - escalation: 升级处理
  - tone_consistency: 语气一致性

knowledge_sources:
  - project_overview.docx
  - faq_document.docx
  - tone_guidelines.docx

presets:
  - name: project_qa
    description: 项目问答
    prompt: |
      请回答关于项目的问题：
      - 问题：{question}
      
      要求：
      1. 只使用提供的文档
      2. 每个回答必须引用来源
      3. 如果信息不明确，说明"待确认"
      4. 敏感问题引导联系负责人
      
  - name: escalation
    description: 升级处理
    prompt: |
      判断问题是否需要升级：
      
      升级条件：
      1. 政策解释
      2. 非公开数据
      3. 未解决的访问问题
      4. 缺失源内容
      5. 合规/法律查询
      
      输出：
      - 是否升级
      - 推荐渠道
      - 所需详细信息

behavior_rules:
  - cite_sources: true
  - no_speculation: true
  - tone_consistency: true
  - escalation_enabled: true

output_formats:
  - markdown: 问答结果
  - json: 结构化数据
```

**教学价值**：
- ✅ Agent 创建完整流程
- ✅ 知识库配置
- ✅ 边界和升级规则
- ✅ 语气一致性控制

**工作量**：16-20 小时
