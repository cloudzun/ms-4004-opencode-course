# 1️⃣ Part 3：Customer Service Analyst（客服分析师）

## 专家方案要点

```
工作流：读取客服工单 CSV ➡️ 提取 Top 3 问题 ➡️ 总结客户情绪 ➡️ 生成 Markdown 周报

OpenCode 开发：
- 编写 summary_skill.md：定义处理单行 CSV 的逻辑和计算满意度的公式

OpenClaw 配置：
- SOUL.md：设定角色为"客户体验洞察官"
- 挂载 file_reader 和 summary_skill
- 配置定时任务：每周五下午 17:00 自动扫描目标文件夹并在群组输出简报
```

---

## 可行性评估

### ✅ 完全可行的部分

| 功能 | 评估 | 说明 |
|------|------|------|
| **读取 CSV/Excel** | ✅ 完全可行 | OpenCode 有 pandas/openpyxl，可读取支持案例数据 |
| **提取 Top 3 问题** | ✅ 完全可行 | 基于案例数量聚合排序，技术简单 |
| **总结客户情绪** | ✅ 可行 | AI 文本分析可生成情绪总结（但非精确情感分析） |
| **生成 Markdown 周报** | ✅ 完全可行 | write 工具支持 Markdown 文件生成 |
| **定时任务（cron）** | ✅ 完全可行 | OpenClaw cron 已验证，支持每周固定时间触发 |
| **SOUL.md 角色设定** | ✅ 完全可行 | OpenClaw 支持 SOUL.md 配置文件 |

---

### ⚠️ 需要澄清/调整的部分

| 功能 | 问题 | 建议方案 |
|------|------|---------|
| **summary_skill.md** | OpenClaw 没有"skill 挂载"概念 | 改为：在 Agent 配置文件中定义提示词模板，或使用 sessions_spawn 调用子 agent |
| **file_reader 挂载** | OpenClaw 没有 file_reader 插件 | 改为：使用 OpenCode read 工具，或在提示词中指定文件路径 |
| **自动扫描文件夹** | cron 支持触发，但"扫描"需额外逻辑 | 改为：cron 触发后，Agent 读取预定义路径的文件，或用户手动触发 |
| **群组输出简报** | 需 message 工具配置 Discord/Telegram | 改为：生成 Markdown 文件，用户自行查看；或配置 message 工具发送 |

---

### ❌ 无法实现的部分

| 功能 | 原因 | 替代方案 |
|------|------|---------|
| **精确满意度计算公式** | 满意度需明确定义（1-5 分？好评率？） | 改为：AI 基于案例内容生成"高/中/低"满意度评估，或从数据中提取评分字段 |
| **实时情感分析 API** | 无 Azure Text Analytics 集成 | 改为：使用 AI 文本分析生成情绪总结（定性而非定量） |
