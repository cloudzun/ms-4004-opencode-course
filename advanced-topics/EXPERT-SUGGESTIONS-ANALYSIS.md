### 8. 💡 最佳实践与避坑指南 ⭐⭐⭐⭐⭐

**Skill 设计避坑**：

| ❌ 常见错误 | ✅ 正确做法 |
|------------|-----------|
| SKILL.md 混合多种功能 | 一个 Skill 只做一件事 |
| 步骤含糊："分析数据" | 步骤具体："统计 Issue Type 列的出现频次，取 Top 3" |
| 缺少输出格式定义 | 明确指定 Markdown 表格 / Excel 列定义 |
| 依赖模型"猜测"文件路径 | 在 SOUL.md 中硬编码默认路径 |
| 要求模型做精确数学计算 | 限定为统计/排名/分布类分析 |

**Agent 编排避坑**：

| ❌ 常见错误 | ✅ 正确做法 |
|------------|-----------|
| 一个 Agent 挂载 5+ 个 Skill | 控制在 2-3 个 Skill，减少误判 |
| SOUL.md 角色描述过于笼统 | 给出具体的身份标签和行为约束 |
| Skill 之间有隐式数据依赖 | 在 SOUL.md 工作流指令中明确串联顺序 |
| 忽略"安全垫" | 法律/财务场景必须有免责声明 |

**教学安全垫设计**：
```markdown
1. 预置脚本兜底：所有工作坊场景的底层解析/搜索/计算脚本由课件组预制
2. 降级策略：
   - 如果学员网络不佳 → 使用本地模型 fallback
   - 如果 OpenClaw 不稳定 → 退回到 OpenCode 单机演示
   - 如果学员进度慢 → 直接发放"参考答案"继续下一步
3. 时间缓冲：每个 Lab 预留 10-15 分钟弹性时间
```

**可直接引用**：✅ 是（强烈推荐！）
