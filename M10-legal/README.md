## 📖 智能办公工作原理

**OpenCode 如何帮你处理法务工作？**

```
你输入自然语言 → OpenCode 理解需求 → 自动编写 Python 代码 → 分析文档/生成报告 → 输出结果
```

**以本模块为例**：

| 步骤 | 你做的 | OpenCode 做的 |
|------|--------|--------------|
| 1 | 输入："请研究 CCPA 和 CPRA 法规" | 理解需求，规划任务 |
| 2 | 等待 | 编写 Python 代码搜索网络 |
| 3 | 等待 | 整理法规信息、生成对比表 |
| 4 | 等待 | 生成法律指南文档 |
| 5 | 收到文件：`legal_guidance.md` | 汇报完成 |

**你需要知道的**：
- ✅ 不需要写代码，用自然语言描述需求即可
- ✅ OpenCode 生成的代码是透明的，你可以查看和学习
- ✅ 如果结果不满意，可以告诉 OpenCode 修改
- ✅ 生成的文件会保存在工作目录
- ✅ **AI 的输出可能有偏差，这是正常的**——你可以像和同事交流一样不断调整

## 4. 最佳实践库

### 合同审查
**标准流程**：
1. 使用标准审查清单
2. 高风险条款必须法务审核
3. 所有修改需版本控制
4. 最终版本存档

**常见风险**：
- ❌ 无限责任条款
- ❌ 模糊的交付要求
- ❌ 缺失的保密条款
- ❌ 不明确的质量标准

**建议**：
- ✅ 使用公司标准模板
- ✅ 明确责任和限制
- ✅ 包含争议解决条款

### 风险评估
**标准流程**：
1. 季度更新风险矩阵
2. 跨部门协作（法务 +IT+HR+ 业务）
3. 记录所有缓解措施
4. 定期审查和报告

**风险分类**：
- 🔴 高优先级（30 天内处理）
- 🟡 中优先级（60 天内处理）
- 🟢 低优先级（90 天内处理）

### 法规研究
**标准流程**：
1. 使用官方来源（.gov 网站）
2. 交叉验证多个来源
3. 标注不确定性
4. 定期更新（法规变更时）

**推荐来源**：
- 加州司法部：https://oag.ca.gov/privacy/ccpa
- 加州隐私保护局：https://cppa.ca.gov/
- 联邦贸易委员会：https://www.ftc.gov/
