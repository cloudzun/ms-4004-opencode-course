# M05 财务场景：使用 OpenCode 提升财务效率

> **适用对象**：财务专员、初级分析师、财务经理  
> **所需工具**：OpenCode Desktop  
> **难度**：⭐⭐ 中等  
> **预计时间**：约 60-90 分钟

你不需要任何编程基础。只需要用自然语言告诉 OpenCode 你想做什么，它会自动编写 Python 代码来完成文件处理任务。

---

## 📖 智能办公工作原理

**OpenCode 如何帮你处理财务工作？**

```
你输入自然语言 → OpenCode 理解需求 → 自动编写 Python 代码 → 分析数据/处理文件 → 输出结果
```

**以本模块为例**：

| 步骤 | 你做的 | OpenCode 做的 |
|------|--------|--------------|
| 1 | 输入："请分析 COGS 数据" | 理解需求，规划任务 |
| 2 | 等待 | 编写 Python 代码读取 Excel |
| 3 | 等待 | 计算关键指标、识别异常值 |
| 4 | 等待 | 生成可视化图表 |
| 5 | 收到文件：`cogs_analysis_report.xlsx` | 汇报完成 |

**你需要知道的**：
- ✅ 不需要写代码，用自然语言描述需求即可
- ✅ OpenCode 生成的代码是透明的，你可以查看和学习
- ✅ 如果结果不满意，可以告诉 OpenCode 修改
- ✅ 生成的文件会保存在 `materials/` 目录
- ✅ **AI 的输出可能有偏差，这是正常的**——你可以像和同事交流一样不断调整

---

## 📁 课件文件

本模块使用以下示例文件（位于 `materials/` 目录）：

| 文件名 | 说明 |
|--------|------|
| `cogs_estimates.xlsx` | EcoSmart 产品线 COGS 估算数据（美元单位） |
| `vendor_contract_a.docx` | 供应商 A（Adatum）合同 |
| `vendor_contract_b.docx` | 供应商 B（Contoso）合同 |

## 实验一：Excel 财务数据分析（COGS 分析）

### 场景描述

你是 Fabrikam 公司的财务分析师。公司正在推出新的 EcoSmart 产品线，运营团队提供了 COGS（销货成本）估算数据。

你需要分析：
- 哪些产品特征或材料驱动了更高的生产成本
- 成本分布和趋势
- 数据中的异常值
- 成本优化建议

---

### 第一步：分析 COGS 数据

打开 OpenCode Desktop，在对话框中输入以下提示词：

---

#### 🟢 入门版提示词（推荐）

```
请读取 materials/cogs_estimates.xlsx，
分析 COGS（销货成本）数据。

要求：
1. 计算总成本、平均成本
2. 找出成本最高的 3 个产品特征
3. 识别数据异常值
4. 提供成本优化建议

重要：
- 保持原始数据的货币单位（美元 USD），不要转换为人民币
- 所有金额前加 $ 符号

将分析结果保存为新的 Excel 文件，
文件名：cogs_analysis_report.xlsx
```

等待 OpenCode 处理完成（约 2-3 分钟）。

---

#### 🟡 优化版提示词（更详细的分析）

<details>
<summary>📋 点击展开优化版提示词</summary>

```
请读取 materials/cogs_estimates.xlsx，
完成以下分析任务：

**分析要求**：
1. 计算关键指标：
   - 总成本、平均成本、成本分布
   - 各产品特征的成本占比
   
2. 识别 Top 5 成本驱动因素：
   - 按平均成本排序
   - 按总成本排序
   
3. 异常值检测：
   - 使用统计方法（如 Z-score > 2 或 平均值±3 倍标准差）
   - 标注异常数据点
   
4. 成本优化建议：
   - 识别可降低成本的领域
   - 提供具体建议

**输出要求**：
- 新建"分析结果"工作表
- 包含分析摘要文字说明
- 包含数据表格
- 生成可视化图表（柱状图、饼图）

**重要**：
- 保持原始数据的货币单位（美元 USD），不要转换为人民币
- 所有金额前加 $ 符号

保存为：cogs_analysis_report.xlsx
```

</details>

---

### 第二步：生成可视化图表（可选）

如果需要更多图表来直观展示分析结果，继续输入：

```
请在 cogs_analysis_report.xlsx 中，
生成以下图表：

1. Top 5 成本驱动因素（柱状图，横轴是产品特征，纵轴是成本金额）
2. 成本分布（饼图，显示各产品成本占比）
3. 异常值检测（使用散点图或高亮标记，标注出高于平均值 3 倍标准差的数据点）

请将图表放在"可视化图表"工作表中。

重要：保持货币单位为美元（USD）。
```

> 💡 **注意**：箱线图（Box Plot）在 Excel 中可能显示不清晰，建议使用**散点图**或**高亮标记**来展示异常值。

---

### 第三步：生成高管摘要（可选）

基于分析结果，生成 Word 格式的高管摘要：

```
请根据 cogs_analysis_report.xlsx 的分析结果，
生成一份高管摘要文档，包含：

1. 关键发现（3-5 个要点）
2. Top 5 成本驱动因素
3. 成本优化建议（Top 3）
4. 风险提示

格式要求：
- 标题字体：微软雅黑，16pt，加粗
- 正文字体：微软雅黑，10.5pt
- 英文字体：Arial

保存为：cogs_executive_summary.docx
```

---

### 预期输出

完成以上步骤后：

| 输出 | 内容 |
|------|------|
| `cogs_analysis_report.xlsx` | COGS 数据分析报告（含图表） |
| `cogs_executive_summary.docx` | 高管摘要（可选） |

> 💡 **注意**：如果 Excel 文件已打开，OpenCode 可能无法覆盖，会生成 `_v2` 版本（见常见问题）。

---

## 实验二：Word 合同对比分析

### 场景描述

你是 Fabrikam 的财务经理。你需要评估两个供应商（Adatum Corporation 和 Contoso, Ltd.）的智能传感器组件合同。

你需要对比：
- 价格条款（单价、折扣、付款条件）
- 交付条款（时间、地点、违约责任）
- 保修条款（期限、范围）
- 法律条款（责任限制、争议解决）
- 风险评估（高/中/低评级）

---

### 第一步：对比两个合同

打开 OpenCode Desktop，在对话框中输入以下提示词：

---

#### 🟢 入门版提示词

```
请对比以下两个供应商合同：
- materials/vendor_contract_a.docx
- materials/vendor_contract_b.docx

对比维度：
1. 价格条款（单价、付款条件）
2. 交付条款（时间、违约责任）
3. 保修条款（期限、范围）
4. 法律条款（责任限制）

格式要求：
- 标题字体：微软雅黑，16pt，加粗
- 正文字体：微软雅黑，10.5pt
- 英文字体：Arial
- 使用表格进行对比

生成对比报告，保存为：contract_comparison.docx
```

---

#### 🟡 优化版提示词（更详细的对比）

<details>
<summary>📋 点击展开优化版提示词</summary>

```
请详细对比以下两个供应商合同：
- vendor_contract_a.docx（Adatum Corporation）
- vendor_contract_b.docx（Contoso, Ltd.）

**对比维度**：
1. **价格条款**
   - 单价
   - 数量折扣
   - 付款条件（Net 30/45/60）
   - 价格调整机制

2. **交付条款**
   - 交付周期
   - 交货地点（FOB/CIF 等）
   - 逾期交付罚则

3. **保修条款**
   - 保修期限
   - 保修范围
   - 响应时间

4. **法律条款**
   - 责任限制
   - 争议解决方式
   - 管辖法律

**输出要求**：
- 并排对比表格
- 差异分析（相同/不同）
- 风险评估（高/中/低）
- 推荐供应商

**格式要求**：
- 标题字体：微软雅黑，16pt，加粗
- 正文字体：微软雅黑，10.5pt
- 英文字体：Arial
- 表格边框清晰

保存为：contract_comparison.docx
```

</details>

---

### 第二步：风险评估（可选）

基于合同对比，进行风险评估：

```
请基于合同对比结果，
对两个合同进行风险评估。

评估维度：
1. 财务风险（30% 权重）
2. 运营风险（25% 权重）
3. 法律风险（20% 权重）
4. 合规风险（15% 权重）
5. 供应商风险（10% 权重）

生成风险矩阵表，
保存为：contract_risk_assessment.docx

格式要求：
- 标题字体：微软雅黑，16pt，加粗
- 正文字体：微软雅黑，10.5pt
```

---

### 第三步：生成谈判建议（可选）

基于对比和风险评估，生成谈判策略：

```
请基于合同对比和风险评估，
生成谈判建议文档，包含：

1. 我方优势（3-5 个）
2. 对方弱点（3-5 个）
3. 建议争取的条款（Top 5，按优先级）
4. 可让步的条款（Top 3）
5. 底线条款（不可让步）

保存为：negotiation_tips.docx

格式要求：
- 标题字体：微软雅黑，16pt，加粗
- 正文字体：微软雅黑，10.5pt
```

---

### 预期输出

完成以上步骤后：

| 输出 | 内容 |
|------|------|
| `contract_comparison.docx` | 合同对比报告 |
| `contract_risk_assessment.docx` | 风险评估报告（可选） |
| `negotiation_tips.docx` | 谈判建议（可选） |

## 附录：提示词技巧

好的提示词包含四个要素，能帮助 OpenCode 给出更准确的结果：

| 要素 | 说明 | 示例 |
|------|------|------|
| **目标** | 你想让 OpenCode 做什么 | "分析 COGS 数据" |
| **背景** | 业务场景、数据说明 | "EcoSmart 新产品线成本估算" |
| **来源** | 数据从哪里来 | "materials/cogs_estimates.xlsx" |
| **要求** | 格式、风格、长度等 | "包含图表、3-5 个建议" |

格式要求：

## ❓ 常见问题

### Q1: Excel 数据读取失败怎么办？

**原因**：文件路径错误或文件被占用。

**解决**：
1. 确认文件路径正确（如 `materials/cogs_estimates.xlsx`）
2. 关闭已打开的 Excel 文件
3. 告诉 OpenCode："请重新读取文件，路径是 materials/cogs_estimates.xlsx"

**预防**：在提示词中明确文件路径。

---

### Q2: Word 文档字体显示为方框怎么办？

**原因**：未明确指定中文字体。

**解决**：在提示词中添加字体要求：

```
格式要求：
- 标题字体：微软雅黑，16pt，加粗
- 正文字体：微软雅黑，10.5pt
- 英文字体：Arial
```

**预防**：每次生成 Word 文档时都包含字体配置。

---

### Q3: Excel 分析结果货币单位变了怎么办？

**原因**：AI 可能自动转换货币单位。

**解决**：在提示词中明确要求：

```
重要：
- 保持原始数据的货币单位（美元 USD），不要转换为人民币
- 所有金额前加 $ 符号
```

---

### Q4: PPT 中文显示为方框怎么办？

**原因**：python-pptx 默认字体不支持中文。

**解决**：在提示词中添加字体配置：

```
使用 python-pptx 库生成 PPT：
- 页面比例：16:9
- 中文字体：微软雅黑
- 英文字体：Arial
```

> 💡 **已优化**：本课程的提示词已包含中文字体配置要求。

---

### Q5: 箱线图显示不正确怎么办？

**原因**：Excel 中箱线图（Box Plot）支持有限，OpenCode 可能生成其他图表。

**解决**：使用更清晰的描述：

```
异常值检测图表：
- 使用散点图或高亮标记
- 标注出高于平均值 3 倍标准差的数据点
```

或者改用 Word/PPT 中展示：

```
在 PPT 中用文字描述异常值：
- 列出异常数据点
- 说明检测方法（平均值±3 倍标准差）
```

---

### Q6: Word 生成报错怎么办？

**原因**：Word 文档结构复杂，生成过程可能报错。

**解决**：
1. **简化输出**：减少复杂表格和格式
2. **改用 Markdown**：实验四和五已改为 Markdown 格式
3. **分段生成**：先内容后格式

**预防**：对于简单文档（会议纪要、邮件），使用 Markdown 格式（`.md`）。

---

### Q7: 会议笔记总结不够准确？

**原因**：笔记内容模糊或 AI 理解有偏差。

**解决**：
1. **补充背景信息**：如"这是 Q4 预算讨论会议"
2. **指定重点**：如"重点关注预算分配和行动项"
3. **迭代优化**：如"请重新总结，行动项需要包含负责人和截止日期"

**示例**：
```
请重新总结会议笔记，
重点关注：
- 预算分配决策
- 行动项（必须包含负责人和截止日期）
- 待讨论议题
```

---

### Q8: 邮件语气不合适？

**原因**：AI 生成的语气可能不符合你的沟通风格。

**解决**：
1. **指定语气**：如"语气要正式/友好/紧急"
2. **提供示例**：如"参考以下邮件风格：[示例邮件]"
3. **多版本对比**：生成多个版本，选择最合适的

**示例**：
```
请重新起草邮件，
语气要求：
- 正式但友好
- 避免过于强硬
- 体现尊重
```
