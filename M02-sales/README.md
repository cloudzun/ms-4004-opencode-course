# M02 销售场景：使用 OpenCode 提升销售效率

> **适用对象**：销售团队、售前工程师、业务拓展人员  
> **所需工具**：OpenCode Desktop（本地安装）  
> **难度**：⭐⭐ 入门  
> **预计时间**：约 45 分钟

你不需要任何编程基础。只需要用自然语言告诉 OpenCode 你想做什么，它会自动编写 Python 代码来完成文件处理任务。

---

## 📖 智能办公工作原理

**OpenCode 如何帮你办公？**

```
你输入自然语言 → OpenCode 理解需求 → 自动编写 Python 代码 → 操作文件 → 输出结果
```

**以本实验为例**：

| 步骤 | 你做的 | OpenCode 做的 |
|------|--------|--------------|
| 1 | 输入："请读取产品文档，生成销售提案" | 理解需求，规划任务 |
| 2 | 等待 | 编写 Python 代码（读取 Word 文档） |
| 3 | 等待 | 分析产品信息、客户案例、价格表 |
| 4 | 等待 | 生成专业的销售提案 Word 文档 |
| 5 | 收到文件：`EcoSense360 销售提案.docx` | 汇报完成 |

**你需要知道的**：
- ✅ 不需要写代码，用自然语言描述需求即可
- ✅ OpenCode 生成的代码是透明的，你可以查看和学习
- ✅ 如果结果不满意，可以告诉 OpenCode 修改（如"请添加 ROI 分析表格"）
- ✅ 生成的文件会保存在 `materials/` 目录
- ✅ **AI 的输出可能有偏差，这是正常的**——你可以像和同事交流一样不断调整，直到满意为止

## 实验一：根据产品文档生成销售提案（Word 文档）

### 场景描述

你是 Fabrikam 公司的销售代表。公司的主力产品 EcoSense 360 是一款智能能源管理解决方案，主要面向酒店和度假村客户。

下周你要参加一个大型酒店集团的采购会议，需要一份专业的销售提案，清晰阐述产品价值、客户痛点、解决方案和投资回报。

你手头有产品技术规格、客户案例和价格表，现在希望用 OpenCode 帮你把这些资料整合成一份专业的销售提案 Word 文档。

## 实验二：分析历史 RFP 数据（Excel）

### 场景描述

作为销售代表，你需要了解过去的 RFP（招标书）响应情况，找出成功和失败的模式，以便在未来的投标中提高胜率。

Fabrikam 的销售运营团队维护着一份历史 RFP 数据记录，包含功能请求、交易规模、输赢结果等信息。你希望用 OpenCode 帮你分析这些数据，找出：
- 客户最常请求的功能有哪些？
- 我们在哪些方面容易输单？
- 哪些区域的胜率最高？

---

### 第一步：分析 RFP 数据趋势

打开 OpenCode Desktop，在对话框中输入：

---

#### 🟢 入门版提示词

```

分析历史 RFP 数据，找出以下趋势：

1. 客户最常请求的 5 个产品功能是什么？
2. 我们在哪些方面容易输单（缺失哪些关键功能）？
3. 哪个区域的胜率最高？

请将分析结果保存到一个新的 Excel 文件中，文件名：RFP 数据分析报告.xlsx
```

## ❓ 常见问题

### Q1: 为什么生成了 `EcoSense360 销售提案_v2.docx` 这样的文件？

**原因**：原文件已被打开或占用，OpenCode 无法覆盖。

**解决**：
1. 关闭已打开的 Word 文档
2. 让 OpenCode 重新生成，或手动将 `_v2` 版本重命名

**预防**：生成文件前先关闭目标文件。

---

### Q2: 图表中的中文显示为方框怎么办？

**原因**：matplotlib 默认字体不支持中文。

**解决**：输入以下提示词让 OpenCode 修复：

```
请在生成图表的 Python 脚本中，添加中文字体配置：

import matplotlib.pyplot as plt
plt.rcParams['font.sans-serif'] = ['Microsoft YaHei', 'SimHei']
plt.rcParams['axes.unicode_minus'] = False

然后重新运行脚本生成报告。
```

> 💡 **已优化**：本课程的提示词已包含中文字体配置要求，第一次生成就应该正常显示中文。

---

### Q3: 提示"Permission denied"或"Device or resource busy"怎么办？

**原因**：Word/Excel 文件已被打开，无法写入。

**解决**：
1. 关闭已打开的文件
2. 让 OpenCode 重新生成

---

### Q4: 生成的内容和格式不符合要求怎么办？

**这是正常的！** OpenCode 是根据你的提示词生成内容的，如果结果不满意：

1. **明确指出问题**：如"ROI 分析表格缺失"
2. **告诉它如何修改**：如"请添加一个表格，展示不同规模酒店的投资回报"
3. **迭代优化**：可能需要 2-3 轮对话才能达到理想效果

**示例**：

| 轮次 | 你的提示词 | 结果 |
|------|-----------|------|
| 第一次 | "请生成一份销售提案" | 只有文字，没有数据表格 |
| 第二次 | "请添加 ROI 分析表格，展示不同规模酒店的投资回报" | 有表格，但数据不完整 |
| 第三次 | "请补充表格数据，包含小型/中型/大型酒店的成本和回报周期" | 完美 ✅ |

> 💡 **鼓励**：不要担心需要多轮对话，这正是 AI 办公的优势——你可以像和同事交流一样不断调整，直到满意为止。

---

### Q5: OpenCode 说"Could not find oldString"怎么办？

**原因**：代码编辑时找不到要修改的内容。

**解决**：让 OpenCode 重新读取文件后重试，或者直接生成新脚本。
