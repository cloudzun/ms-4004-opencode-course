# M08 沟通场景：使用 OpenCode 提升沟通效率

> **适用对象**：沟通专员、内部沟通经理、公关专员  
> **所需工具**：OpenCode Desktop  
> **难度**：⭐⭐ 中等  
> **预计时间**：约 45-60 分钟

你不需要任何编程基础。只需要用自然语言告诉 OpenCode 你想做什么，它会自动编写 Python 代码来完成文件处理任务。

---

## 📖 智能办公工作原理

**OpenCode 如何帮你处理沟通工作？**

```
你输入自然语言 → OpenCode 理解需求 → 自动编写 Python 代码 → 分析数据/处理文件 → 输出结果
```

**以本模块为例**：

| 步骤 | 你做的 | OpenCode 做的 |
|------|--------|--------------|
| 1 | 输入："请生成战略沟通简报" | 理解需求，规划任务 |
| 2 | 等待 | 编写 Python 代码读取产品文档 |
| 3 | 等待 | 生成战略简报内容 |
| 4 | 等待 | 保存为 Word 文档 |
| 5 | 收到文件：`communication_brief.docx` | 汇报完成 |

**你需要知道的**：
- ✅ 不需要写代码，用自然语言描述需求即可
- ✅ OpenCode 生成的代码是透明的，你可以查看和学习
- ✅ 如果结果不满意，可以告诉 OpenCode 修改
- ✅ 生成的文件会保存在 `materials/` 目录
- ✅ **AI 的输出可能有偏差，这是正常的**——你可以像和同事交流一样不断调整

## 实验一：Word 战略沟通简报

### 场景描述

你是 Boulder Innovations 公司的沟通经理。公司即将推出新的内部沟通平台 AuroraHub，你需要为高管团队创建一份战略沟通简报。

你需要：
- 介绍 AuroraHub 平台及其目的
- 说明关键特性和优势
- 展示与公司战略目标的对齐
- 描述预期业务影响

---

### 第一步：生成战略沟通简报

打开 OpenCode Desktop，在对话框中输入以下提示词：

---

#### 🟢 入门版提示词（推荐）

```
请读取 materials/product_overview.docx，
生成一份战略沟通简报。

**输出结构**：

# 战略沟通简报

## 1. 执行摘要（200-300 字）
- 项目概述
- 核心价值主张

## 2. 产品/项目介绍
- 是什么
- 为什么需要
- 目标受众

## 3. 关键特性和优势
- 特性 1 + 对应优势
- 特性 2 + 对应优势
- 特性 3 + 对应优势

## 4. 与战略目标对齐
- 如何支持公司战略
- 预期业务影响

## 5. 建议下一步
- 短期行动（1 个月内）
- 中期行动（1-3 个月）

保存为：communication_brief.docx

**格式要求**：
- 标题字体：微软雅黑，16pt，加粗
- 正文字体：微软雅黑，10.5pt
- 英文字体：Arial
```

等待 OpenCode 处理完成（约 2-3 分钟）。

## 实验二：Excel 沟通指标分析

### 场景描述

你是 Boulder Innovations 公司的沟通经理。你需要分析 Q4 季度邮件通讯的参与度数据，识别高/低参与度部门，为改进沟通策略提供数据支持。

你需要：
- 计算平均打开率、平均点击率
- 识别参与度最高/最低的 3 个部门
- 生成可视化图表
- 提供改进建议

---

### 第一步：分析沟通指标

打开 OpenCode Desktop，在对话框中输入以下提示词：

---

#### 🟢 入门版提示词（推荐）

```
请读取 materials/newsletter_metrics.xlsx，
分析沟通指标数据。

**数据列名**：
- Department（部门）
- Emails sent（发送数量）
- Emails opened（打开数量）
- Links clicked（点击数量）
- Open rate (%)（打开率）
- Click-through rate (%)（点击率）

**要求**：
1. 计算平均打开率、平均点击率
2. 识别参与度最高/最低的 3 个部门
3. 生成对比图表（柱状图）
4. 提供改进建议（3-5 条）

保存为：communication_analysis.xlsx
```

等待 OpenCode 处理完成（约 2-3 分钟）。

---

## ❓ 常见问题

### Q0: 提示"FileNotFoundError"或"找不到文件"怎么办？

**原因**：OpenCode 不知道文件的完整路径，或者工作目录设置不正确。

**解决**：

**方法 1：使用完整路径（推荐）**
```
请读取 C:/Users/你的用户名/Documents/M08-communications/materials/product_overview.docx，
生成战略沟通简报。

保存为：C:/Users/你的用户名/Documents/M08-communications/communication_brief.docx
```

**方法 2：确认工作目录**
1. 打开 OpenCode Desktop
2. 检查工作目录是否指向 `M08-communications/` 目录
3. 如果不是，重新打开 OpenCode 并选择正确的目录

**方法 3：使用相对路径**
```
请读取 materials/product_overview.docx，
生成战略沟通简报。

保存为：communication_brief.docx
```
前提：OpenCode 的工作目录必须是 `M08-communications/` 的父目录

**预防**：
- ✅ 在提示词中使用完整路径（最可靠）
- ✅ 确认 OpenCode 工作目录正确
- ✅ 先在文件资源管理器中确认文件位置

### Q1: 战略沟通简报与一般报告有什么区别？

**原因**：战略沟通简报强调业务影响和战略对齐。

**区别**：
- **一般报告**：描述事实、数据、进展
- **战略简报**：强调战略对齐、业务影响、建议决策

**解决**：在提示词中明确要求：
```
**重点**：
- 突出战略对齐
- 量化业务影响
- 提供明确建议
```

---

### Q2: 沟通指标（打开率/点击率）如何计算？

**打开率** = 打开邮件人数 ÷ 发送总数 × 100%

**点击率** = 点击链接人数 ÷ 打开邮件人数 × 100%

**示例**：
- 发送 100 封邮件
- 60 人打开 → 打开率 = 60%
- 15 人点击 → 点击率 = 15 ÷ 60 = 25%

---

### Q3: 全员公告邮件的语气如何把握？

**原因**：AI 生成的语气可能不符合公司文化。

**解决**：
1. **指定语气**：如"语气要正式/友好/紧急"
2. **提供示例**：如"参考以下邮件风格：[示例邮件]"
3. **多版本对比**：生成多个版本，选择最合适的
