# M09 客服场景：使用 OpenCode 提升客服效率

> **适用对象**：客服专员、客服经理、支持团队  
> **所需工具**：OpenCode Desktop  
> **难度**：⭐⭐ 中等  
> **预计时间**：约 45-60 分钟

你不需要任何编程基础。只需要用自然语言告诉 OpenCode 你想做什么，它会自动编写 Python 代码来完成文件处理任务。

---

## 📖 智能办公工作原理

**OpenCode 如何帮你处理客服工作？**

```
你输入自然语言 → OpenCode 理解需求 → 自动编写 Python 代码 → 分析数据/处理文件 → 输出结果
```

**以本模块为例**：

| 步骤 | 你做的 | OpenCode 做的 |
|------|--------|--------------|
| 1 | 输入："请生成客户支持旅程映射" | 理解需求，规划任务 |
| 2 | 等待 | 编写 Python 代码生成支持流程 |
| 3 | 等待 | 分类整理支持步骤 |
| 4 | 等待 | 识别客户痛点 |
| 5 | 收到文件：`support_journey.md` | 汇报完成 |

**你需要知道的**：
- ✅ 不需要写代码，用自然语言描述需求即可
- ✅ OpenCode 生成的代码是透明的，你可以查看和学习
- ✅ 如果结果不满意，可以告诉 OpenCode 修改
- ✅ 生成的文件会保存在 `materials/` 目录
- ✅ **AI 的输出可能有偏差，这是正常的**——你可以像和同事交流一样不断调整

---

## 📁 课件文件

本模块使用以下示例文件（位于 `materials/` 目录）：

| 文件名 | 说明 |
|--------|------|
| `support_case_summaries.docx` | 支持案例摘要 |
| `support_cases_90days.xlsx` | 90 天支持案例数据 |
| `support_meeting_notes.docx` | 支持团队会议笔记 |

> 请将 `materials/` 文件夹放在你的工作目录下，然后打开 OpenCode Desktop，将工作目录指向该文件夹的上一级目录。

## 实验一：客户支持旅程映射

### 场景描述

你是 Lamna Healthcare 公司的客服经理。公司需要设计理想的客户支持旅程，从客户报告问题到问题解决的完整流程。

你需要：
- 生成支持流程步骤（12-15 个）
- 按阶段分类（受理/处理/解决/跟进）
- 识别客户痛点（8-10 个）
- 生成流程总结

---

### 第一步：生成支持旅程映射

打开 OpenCode Desktop，在对话框中输入以下提示词：

---

#### 🟢 入门版提示词（推荐）

```
请为医疗产品客户支持生成支持旅程映射。

**背景**：
- 公司：Lamna Healthcare
- 产品：医疗预约管理平台
- 客户：诊所和医疗机构

**输出要求**：

# 客户支持旅程映射

## 1. 支持流程步骤（12-15 个）
按阶段分组：
- 受理阶段
- 处理阶段
- 解决阶段
- 跟进阶段

## 2. 客户痛点识别（8-10 个）
- 每个阶段的主要痛点
- 痛点影响分析

## 3. 流程总结
200-300 字摘要

保存为：support_journey.md
```

等待 OpenCode 处理完成（约 2-3 分钟）。

## 实验二：客户回复模板

### 场景描述

你是 Tailwind Traders 公司的客服经理。公司需要创建标准客户回复模板，确保客服团队回复的一致性和专业性。

你需要：
- 生成常见问题回复模板
- 每个问题生成 2 个版本（正式版/友好版）
- 包含关键信息和行动呼吁

---

### 第一步：生成回复模板

打开 OpenCode Desktop，在对话框中输入以下提示词：

---

#### 🟢 入门版提示词（推荐）

```
请创建客户回复模板，
包含以下常见问题：

1. 产品退换货政策
2. 配送时间查询
3. 产品质量问题
4. 订单状态查询

每个问题生成 2 个版本：
1. 正式版（专业、详细，150-200 字）
2. 友好版（亲切、简洁，100-150 字）

保存为：response_templates.docx

**格式要求**：
- 标题字体：微软雅黑，16pt，加粗
- 正文字体：微软雅黑，10.5pt
- 英文字体：Arial
```

---

#### 🟡 优化版提示词（更详细）

<details>
<summary>📋 点击展开优化版提示词</summary>

```
请创建客户回复模板，
用于 Tailwind Traders 客服团队。

**背景**：
- 公司：Tailwind Traders
- 受众：B2B 经销商
- 语气：专业但友好

**常见问题**：

### 1. 产品退换货政策
- 退货期限
- 退货条件
- 退款流程

### 2. 配送时间查询
- 标准配送时间
- 加急配送选项
- 配送追踪

### 3. 产品质量问题
- 质量问题定义
- 报告流程
- 解决方案

### 4. 订单状态查询
- 如何查询订单
- 订单处理时间
- 延迟处理

**要求**：
- 每个问题生成 2 个版本：
  1. 正式版（专业、详细，150-250 字）
  2. 友好版（亲切、简洁，100-200 字）
- 包含关键信息
- 包含行动呼吁（下一步操作）
- 包含联系人信息

保存为：response_templates.docx

**格式要求**：
- 标题字体：微软雅黑，16pt，加粗
- 正文字体：微软雅黑，10.5pt
- 英文字体：Arial
- 使用表格对比版本
```

</details>

---

## 实验三：故障排查清单

### 场景描述

你是 Tailwind Traders 公司的客服经理。经销商经常报告产品连接问题，你需要创建标准化的故障排查清单，帮助客服团队和经销商快速诊断和解决问题。

你需要：
- 生成初步诊断步骤（5-7 个）
- 生成详细排查步骤（10-12 个）
- 创建检查清单（证据要求）
- 定义升级规则

---

### 第一步：生成故障排查清单

打开 OpenCode Desktop，在对话框中输入以下提示词：

---

#### 🟢 入门版提示词（推荐）

```
请为产品连接问题创建故障排查清单。

**背景**：
- 公司：Tailwind Traders
- 产品：户外设备
- 受众：B2B 经销商

**包含**：
1. 初步诊断步骤（5-7 个）
2. 详细排查步骤（10-12 个）
3. 检查清单（经销商需提供的证据）
4. 升级规则（何时升级给技术支持）

保存为：troubleshooting_checklist.md
```

等待 OpenCode 处理完成（约 2-3 分钟）。

---

#### 🟡 优化版提示词（更详细）

<details>
<summary>📋 点击展开优化版提示词</summary>

```
请为产品连接问题创建详细的故障排查清单。

**背景**：
- 公司：Tailwind Traders
- 产品：户外设备（电动工具、园艺设备等）
- 受众：B2B 经销商
- 问题类型：产品连接问题（蓝牙/WiFi/物理连接）

**输出结构**：

# 产品连接问题故障排查清单

## 1. 初步诊断（5-7 个步骤）
每个步骤包含：
- 步骤描述
- 预计时间
- 通过/失败标准

## 2. 详细排查（10-12 个步骤）
按问题类型分组：
- 蓝牙连接问题
- WiFi 连接问题
- 物理连接问题

## 3. 检查清单（证据要求）
经销商必须提供：
- 产品照片
- 购买凭证
- 序列号照片
- 问题视频（可选）

## 4. 常见解决方案
- 解决方案 1 + 适用场景
- 解决方案 2 + 适用场景
- 临时变通方案

## 5. 升级规则
何时升级给技术支持：
- 安全相关问题
- 重复失败（3 次以上）
- 出厂缺陷
- 保修相关问题

## 6. 联系信息
- 技术支持邮箱
- 技术支持电话
- 服务时间

保存为：troubleshooting_checklist.md
```

</details>

## ❓ 常见问题

### Q0: 提示"FileNotFoundError"或"找不到文件"怎么办？

**原因**：OpenCode 不知道文件的完整路径，或者工作目录设置不正确。

**解决**：

**方法 1：使用完整路径（推荐）**
```
请读取 C:/Users/你的用户名/Documents/M09-customer-service/materials/support_case_summaries.docx，
生成支持旅程映射。

保存为：C:/Users/你的用户名/Documents/M09-customer-service/support_journey.md
```

**方法 2：确认工作目录**
1. 打开 OpenCode Desktop
2. 检查工作目录是否指向 `M09-customer-service/` 目录
3. 如果不是，重新打开 OpenCode 并选择正确的目录

**方法 3：使用相对路径**
```
请读取 materials/support_case_summaries.docx，
生成支持旅程映射。

保存为：support_journey.md
```
前提：OpenCode 的工作目录必须是 `M09-customer-service/` 的父目录

**预防**：
- ✅ 在提示词中使用完整路径（最可靠）
- ✅ 确认 OpenCode 工作目录正确
- ✅ 先在文件资源管理器中确认文件位置

### Q1: 客户支持旅程与一般流程有什么区别？

**原因**：客户支持旅程强调客户体验和痛点。

**区别**：
- **一般流程**：描述内部步骤
- **支持旅程**：强调客户视角、痛点、体验

**解决**：在提示词中明确要求：
```
**重点**：
- 从客户视角描述
- 识别每个阶段的痛点
- 提供改进建议
```

---

### Q2: 回复模板的语气如何把握？

**原因**：AI 生成的语气可能不符合公司文化。

**解决**：
1. **指定语气**：如"语气要正式/友好/紧急"
2. **提供示例**：如"参考以下邮件风格：[示例邮件]"
3. **多版本对比**：生成多个版本，选择最合适的

---

### Q3: 故障排查清单不够详细怎么办？

**原因**：提示词不够具体，或者 AI 理解有偏差。

**解决**：
1. **明确要求数量**：如"生成 10-12 个详细排查步骤"
2. **提供约束**：如"按问题类型分组（蓝牙/WiFi/物理）"
3. **迭代优化**：如"请扩展第 3 步，添加更多细节"
