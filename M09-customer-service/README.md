# M09 客服场景：使用 OpenCode 提升客服效率

> **适用对象**：客服专员、客服经理、支持团队  
> **所需工具**：OpenCode Desktop  
> **难度**：⭐⭐ 中等  
> **预计时间**：约 60-90 分钟

你不需要任何编程基础。只需要用自然语言告诉 OpenCode 你想做什么，它会自动编写 Python 代码来完成文件处理任务。

---

## 📖 智能办公工作原理

**OpenCode 如何帮你处理客服工作？**

```
你输入自然语言 → OpenCode 理解需求 → 自动编写 Python 代码 → 分析数据/处理文件 → 输出结果
```

**以本模块为例**：

| 步骤 | 你做的 | OpenCode 做的 |
|------|--------|--------------|
| 1 | 输入："请分析 90 天支持案例数据" | 理解需求，规划任务 |
| 2 | 等待 | 编写 Python 代码读取 Excel |
| 3 | 等待 | 计算关键指标、识别 Top 问题 |
| 4 | 等待 | 生成分析仪表板 |
| 5 | 收到文件：`support_dashboard.md` | 汇报完成 |

**你需要知道的**：
- ✅ 不需要写代码，用自然语言描述需求即可
- ✅ OpenCode 生成的代码是透明的，你可以查看和学习
- ✅ 如果结果不满意，可以告诉 OpenCode 修改
- ✅ 生成的文件会保存在 `materials/` 目录
- ✅ **AI 的输出可能有偏差，这是正常的**——你可以像和同事交流一样不断调整

## 实验一：支持数据分析

### 场景描述

作为客服改进项目的第一步，你需要分析过去 90 天的支持案例数据，识别 Top 问题、趋势和关键洞察。这些数据将指导后续的所有改进工作。

你需要：
- 分析 90 天支持案例数据
- 识别 Top 5 问题类别
- 分析严重程度分布和解决时间
- 计算升级率
- 生成关键洞察和建议

---

### 第一步：分析支持案例数据

打开 OpenCode Desktop，在对话框中输入以下提示词：

---

#### 🟢 入门版提示词（推荐）

```
请读取 materials/support_cases_90days.xlsx，
分析 90 天支持案例数据。

**要求**：
1. Top 5 问题类别（按出现频率）
2. 严重程度分布（低/中/高/紧急）
3. 平均解决时间
4. 升级率分析
5. 关键洞察和建议（3-5 条）

保存为：support_dashboard.md
```

等待 OpenCode 处理完成（约 2-3 分钟）。

## ❓ 常见问题

### Q0: 提示"FileNotFoundError"或"找不到文件"怎么办？

**原因**：OpenCode 不知道文件的完整路径，或者工作目录设置不正确。

**解决**：

**方法 1：使用完整路径（推荐）**
```
请读取 C:/Users/你的用户名/Documents/M09-customer-service/materials/support_cases_90days.xlsx，
分析 90 天支持案例数据。

保存为：C:/Users/你的用户名/Documents/M09-customer-service/support_dashboard.md
```

**方法 2：确认工作目录**
1. 打开 OpenCode Desktop
2. 检查工作目录是否指向 `M09-customer-service/` 目录
3. 如果不是，重新打开 OpenCode 并选择正确的目录

**方法 3：使用相对路径**
```
请读取 materials/support_cases_90days.xlsx，
分析 90 天支持案例数据。

保存为：support_dashboard.md
```
前提：OpenCode 的工作目录必须是 `M09-customer-service/` 的父目录

**预防**：
- ✅ 在提示词中使用完整路径（最可靠）
- ✅ 确认 OpenCode 工作目录正确
- ✅ 先在文件资源管理器中确认文件位置

---

### Q1: 数据分析结果不够详细怎么办？

**原因**：提示词不够具体，或者 AI 理解有偏差。

**解决**：
1. **明确要求数量**：如"生成 Top 5 问题类别"
2. **指定分析维度**：如"按问题类别、严重程度、解决时间分析"
3. **迭代优化**：如"请深入分析升级率，按问题类别分组"

---

### Q2: 回复模板的语气如何把握？

**原因**：AI 生成的语气可能不符合公司文化。

**解决**：
1. **指定语气**：如"语气要正式/友好/紧急"
2. **提供示例**：如"参考以下邮件风格：[示例邮件]"
3. **多版本对比**：生成多个版本，选择最合适的

---

### Q3: 故障排查清单不够详细怎么办？

**原因**：提示词不够具体，或者 AI 理解有偏差。

**解决**：
1. **明确要求数量**：如"生成 10-12 个详细排查步骤"
2. **提供约束**：如"按问题类型分组（基于 Top 问题）"
3. **迭代优化**：如"请扩展第 3 步，添加更多细节"
