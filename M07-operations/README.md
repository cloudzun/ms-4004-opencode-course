# M07 运营场景：使用 OpenCode 提升运营效率

> **适用对象**：运营专员、项目经理、运营通才  
> **所需工具**：OpenCode Desktop  
> **难度**：⭐⭐ 中等  
> **预计时间**：约 60-90 分钟

你不需要任何编程基础。只需要用自然语言告诉 OpenCode 你想做什么，它会自动编写 Python 代码来完成文件处理任务。

---

## 📖 智能办公工作原理

**OpenCode 如何帮你处理运营工作？**

```
你输入自然语言 → OpenCode 理解需求 → 自动编写 Python 代码 → 分析数据/处理文件 → 输出结果
```

**以本模块为例**：

| 步骤 | 你做的 | OpenCode 做的 |
|------|--------|--------------|
| 1 | 输入："请为项目生成头脑风暴笔记" | 理解需求，规划任务 |
| 2 | 等待 | 编写 Python 代码生成创意 |
| 3 | 等待 | 分类整理想法 |
| 4 | 等待 | 生成结构化文档 |
| 5 | 收到文件：`brainstorming_notes.md` | 汇报完成 |

**你需要知道的**：
- ✅ 不需要写代码，用自然语言描述需求即可
- ✅ OpenCode 生成的代码是透明的，你可以查看和学习
- ✅ 如果结果不满意，可以告诉 OpenCode 修改
- ✅ 生成的文件会保存在 `materials/` 目录
- ✅ **AI 的输出可能有偏差，这是正常的**——你可以像和同事交流一样不断调整

---

## 实验一：项目头脑风暴

### 场景描述

你是 Adatum 公司的运营经理。公司计划安装新的锅炉供暖系统，你需要组织一次头脑风暴会议，生成项目步骤建议并分类整理。

你需要：
- 生成项目步骤建议（12-15 个）
- 生成风险缓解建议（6-8 个）
- 按阶段分类整理
- 生成项目总结

---

### 第一步：生成头脑风暴笔记

打开 OpenCode Desktop，在对话框中输入以下提示词：

---

#### 🟢 入门版提示词（推荐）

```
请为办公室锅炉安装项目生成头脑风暴笔记。

输出结构：
1. 项目步骤建议（12-15 个，按阶段分组）
2. 风险缓解建议（6-8 个，包含缓解措施）
3. 关键注意事项
4. 项目总结（200-300 字）

保存为：brainstorming_notes.md
```

等待 OpenCode 处理完成（约 2-3 分钟）。

## 实验二：研究报告生成

### 场景描述

你是 Adatum 公司的运营经理。你需要研究锅炉 vs 炉供暖系统的差异，生成一份对比报告供管理层决策参考。

你需要：
- 基于网络研究生成报告
- 对比两种系统的特点
- 分析成本和维护差异
- 提供建议

---

### 第一步：生成供暖系统对比报告（双路径选择）

> 💡 **说明**：本实验提供两种工作路径，请根据你的实际情况选择：
> - **入门版**：从零开始研究（适合没有现成报告时）
> - **优化版**：基于现有报告生成摘要（适合已有详细报告时）

---

#### 🟢 入门版：从零开始研究（推荐）

**场景**：你没有现成的供暖系统对比报告，需要从头研究。

```
请研究并生成一份供暖系统对比报告。

**研究要求**：
- 使用网络搜索获取最新信息
- 在报告中标注信息来源（如"根据 XX 网站数据"）
- 列出参考的网站或资料

**对比内容**：
1. 锅炉系统 vs 炉系统
2. 工作原理
3. 能效对比
4. 成本分析
5. 维护要求
6. 适用场景

保存为：heating_system_comparison.md
```

> 💡 **注意**：OpenCode 会根据你的提示词自动调用内置的 `web_search` 功能进行网络研究。请观察执行过程，生成的报告中会包含信息来源标识。在后续的高阶课程中，我们会实现增强的互联网搜索功能，支持更多数据源和更精确的检索。

---

#### 🟡 优化版：基于现有报告生成高管摘要

**场景**：你已有一份详细的供暖系统对比报告（如 `materials/heating_systems_report.docx`），现在需要为高管生成简洁的摘要版本。

<details>
<summary>📋 点击展开优化版提示词</summary>

```

生成一份高管摘要报告。

**场景说明**：
- 你已有一份详细的供暖系统对比报告
- 现在需要为高管生成简洁的摘要版本
- 重点突出关键对比点和建议

**包含内容**：
1. 执行摘要（200-300 字）
2. 关键对比表格
   - 锅炉系统特点
```

</details>

## 实验四：项目跟踪文档

### 场景描述

> 💡 **说明**：本实验的"配送中心扩建项目"是为了演示 RAID/RACI 专业框架而设计的**示例场景**，并非真实项目。原版 MS-4004 课程中也没有对应的原型项目。我们的重点是通过这个场景学习：
> - RAID 日志（风险/假设/问题/依赖）的使用方法
> - RACI 矩阵（责任分配）的构建流程
> - 项目跟踪文档的完整结构
> 
> 你可以将这些框架应用到任何真实项目中。

你负责一个配送中心扩建项目，需要创建项目跟踪文档，跟踪里程碑、风险、责任和决策。

你需要创建：
1. 里程碑和时间线
2. 风险日志（RAID 框架）
3. 责任矩阵（RACI 框架）
4. 决策日志
5. 变更日志

---

### 第一步：创建项目跟踪文档

打开 OpenCode Desktop，在对话框中输入以下提示词：

## ❓ 常见问题

### Q0: 提示"FileNotFoundError"或"找不到文件"怎么办？

**原因**：OpenCode 不知道文件的完整路径，或者工作目录设置不正确。

**解决**：

**方法 1：使用完整路径（推荐）**
```
请读取 C:/Users/你的用户名/Documents/M07-operations/materials/heating_systems_report.docx，
生成高管摘要报告。

保存为：C:/Users/你的用户名/Documents/M07-operations/heating_executive_summary.docx
```

**方法 2：确认工作目录**
1. 打开 OpenCode Desktop
2. 检查工作目录是否指向 `M07-operations/` 目录
3. 如果不是，重新打开 OpenCode 并选择正确的目录

**方法 3：使用相对路径**
```
请读取 materials/heating_systems_report.docx，
生成高管摘要报告。

保存为：heating_executive_summary.docx
```
前提：OpenCode 的工作目录必须是 `M07-operations/` 的父目录

**预防**：
- ✅ 在提示词中使用完整路径（最可靠）
- ✅ 确认 OpenCode 工作目录正确
- ✅ 先在文件资源管理器中确认文件位置

### Q1: 头脑风暴想法不够多怎么办？

**原因**：提示词不够开放，或者 AI 理解有偏差。

**解决**：
1. **明确要求数量**：如"生成 15-20 个想法"
2. **分多次生成**：如"再生成 10 个关于风险的想法"
3. **提供示例**：如"类似：风险评估、供应商筛选、时间规划..."

---

### Q2: 项目跟踪文档太复杂怎么办？

**原因**：RAID/RACI 框架对新手可能复杂。

**解决**：
1. **简化版本**：先创建简单的里程碑列表
2. **分步创建**：先里程碑，再风险，再责任
3. **使用模板**：让 OpenCode 提供填空式模板

---

### Q3: 邮件语气不合适？

**原因**：AI 生成的语气可能不符合你的沟通风格。

**解决**：
1. **指定语气**：如"语气要正式/友好/紧急"
2. **提供示例**：如"参考以下邮件风格：[示例邮件]"
3. **多版本对比**：生成多个版本，选择最合适的
