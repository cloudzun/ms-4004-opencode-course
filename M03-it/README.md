# M03 IT 场景：使用 OpenCode 提升 IT 项目管理效率

> **适用对象**：IT 项目经理、技术负责人、系统管理员  
> **所需工具**：OpenCode Desktop（本地安装）  
> **难度**：⭐⭐⭐ 中等  
> **预计时间**：约 60 分钟

你不需要任何编程基础。只需要用自然语言告诉 OpenCode 你想做什么，它会自动编写 Python 代码来完成文件处理任务。

---

## 📖 智能办公工作原理

**OpenCode 如何帮你办公？**

```
你输入自然语言 → OpenCode 理解需求 → 自动编写 Python 代码 → 操作文件 → 输出结果
```

**以本实验为例**：

| 步骤 | 你做的 | OpenCode 做的 |
|------|--------|--------------|
| 1 | 输入："请读取项目文档，生成项目状态报告" | 理解需求，规划任务 |
| 2 | 等待 | 编写 Python 代码（读取 Word 文档） |
| 3 | 等待 | 分析项目计划、状态更新、调查结果 |
| 4 | 等待 | 生成专业的项目状态报告 Word 文档 |
| 5 | 收到文件：`项目状态报告.docx` | 汇报完成 |

**你需要知道的**：
- ✅ 不需要写代码，用自然语言描述需求即可
- ✅ OpenCode 生成的代码是透明的，你可以查看和学习
- ✅ 如果结果不满意，可以告诉 OpenCode 修改（如"请添加风险分析章节"）
- ✅ 生成的文件会保存在 `materials/` 目录
- ✅ **AI 的输出可能有偏差，这是正常的**——你可以像和同事交流一样不断调整，直到满意为止

## 实验一：根据项目文档生成项目状态报告（Word 文档）

### 场景描述

你是 Contoso 公司的 IT 项目经理。公司正在进行多个 IT 项目，包括 CipherGuard 安全系统部署和 Expansion 系统扩展项目。

下周你要向高级管理层汇报项目进展，需要一份专业的项目状态报告，清晰阐述项目进度、风险、问题和下一步计划。

你手头有项目计划书、项目概述和状态总结，现在希望用 OpenCode 帮你把这些资料整合成一份专业的项目状态报告 Word 文档。

---

### 第一步：生成项目状态报告

打开 OpenCode Desktop，在对话框中输入以下提示词：

> 💡 **提示**：直接复制下方文字粘贴到 OpenCode 的对话框中，然后按回车发送。

---

#### 🟢 入门版提示词（推荐先试这个）

```
请读取 materials 目录下的以下文件：
- Contoso CipherGuard project plan.docx（项目计划）
- Contoso_Expansion_Project_Overview.docx（项目概述）
- Fabrikam Q1 Project Status Summary.docx（状态总结）

以 IT 项目经理的身份，为高级管理层创建一份项目状态报告。

报告内容要求：
1. 标题：IT 项目状态报告（2026 年第一季度）
2. 执行摘要：概述整体项目进展（2-3 段）
3. 项目进度：各项目的里程碑完成情况（使用表格）
4. 风险与问题：主要风险和已识别问题
5. 下一步计划：下阶段的关键任务和交付物

请将报告保存为 Word 文档，文件名：项目状态报告.docx
```

等待 OpenCode 处理完成，通常需要 1-2 分钟。

---

#### 🟡 如果效果不满意，可以这样优化

**常见问题及优化提示词**：

| 缺少具体数据 | "请补充具体数据，如完成百分比、预计日期、负责人" |
| 内容太简单 | "请补充更多细节，每个章节至少 5 个要点" |
| 生成英文内容 | "请使用简体中文重新生成，专业术语保留英文原文" |

**或者直接使用优化版提示词**：

- Contoso_Expansion_Project_Overview.docx（项目概述）

以 IT 项目经理的身份，为高级管理层创建一份项目状态报告。

### 第二步：根据调查结果补充用户反馈

查看生成的报告后，你希望加入用户反馈内容。继续输入：

```
请读取 materials/Project_Nexus_survey_results.xlsx 文件，

- 调查参与情况（参与人数、响应率）
- 整体满意度评分

- 主要改进建议（至少 3 条）
- 后续行动计划

请更新并保存文档，保持原有格式不变。
```

> 💡 **注意**：如果提示文件被占用（Permission denied），请先关闭已打开的 Word 文档，然后让 OpenCode 重新生成。

## 实验二：分析项目调查数据（Excel）

### 场景描述

作为 IT 项目经理，你需要了解项目 Nexus 的用户反馈和采用情况，找出改进方向，以便在下一阶段优化项目管理。

项目 Nexus 调查团队维护着一份调查结果数据，包含用户满意度、功能评价、改进建议等信息。你希望用 OpenCode 帮你分析这些数据，找出：
- 用户最满意的功能有哪些？
- 用户反馈的主要问题是什么？
- 哪些功能需要优先改进？

---

### 第一步：分析调查数据趋势

打开 OpenCode Desktop，在对话框中输入：

---

#### 🟢 入门版提示词

```
请读取 materials/Project_Nexus_survey_results.xlsx 文件，
分析项目 Nexus 的调查结果数据，找出以下趋势：

1. 用户满意度评分（平均分、最高分、最低分）
2. 用户最满意的前 3 个功能
3. 用户反馈最多的前 3 个问题

请将分析结果保存到一个新的 Excel 文件中，文件名：调查数据分析报告.xlsx
```

---

#### 🟡 优化版提示词（需要可视化时用）

<details>
<summary>📋 点击展开优化版提示词</summary>

```
请读取 materials/Project_Nexus_survey_results.xlsx 文件，
分析项目 Nexus 的调查结果数据，完成以下任务：

1. 计算用户满意度统计（平均分、最高分、最低分、标准差）
2. 统计用户最满意的前 3 个功能
3. 统计用户反馈最多的前 3 个问题

   - 分析摘要（文字说明）
   - 满意度统计表

技术要求：
- 使用 openpyxl 操作 Excel 文件

- 保存为 Excel 文件，文件名：调查数据分析报告.xlsx

3. 优先级 3 改进项（影响小、实施容易）

请用表格形式展示，包含改进项、优先级、预计工时、负责人建议。
```

## 实验三：根据项目报告生成 PPT 演讲大纲（Word 格式）

> ⚠️ **技术说明**：PPT 生成是一个技术挑战较大的任务。当前方案采用"曲线救国"策略：
> 1. OpenCode 生成 PPT 内容大纲和演讲备注（保存为 Word）
> 2. 用户手动复制到 PowerPoint，或使用其他工具转换
> 3. 随着模型和 skill 的进步，未来可能支持直接生成 PPT 文件

### 场景描述

你是 Contoso 公司的 IT 项目经理。刚刚完成了项目状态报告，现在需要向高级管理层进行汇报演示。

你需要一份 PPT 演示文稿，清晰展示项目进展、风险、问题和下一步计划。你希望用 OpenCode 帮你根据项目状态报告生成 PPT 内容大纲和演讲备注。

---

### 第一步：生成 PPT 演讲大纲

打开 OpenCode Desktop，在对话框中输入以下提示词：

---

#### 🟢 入门版提示词

生成一份 PPT 演示文稿的演讲大纲。

2. 每张幻灯片包含：
   - 幻灯片标题

   - 演讲备注（详细讲解内容）
3. 结构清晰，适合向高级管理层汇报

请将大纲保存为 Word 文档，文件名：PPT 演讲大纲.docx

### 第三步：转换为 PPT（手动或自动）

**方案 A：使用 Node.js + pptxgenjs 生成（推荐，真正的.pptx）**

使用 Anthropic 官方的 pptx skill，通过 Node.js 和 pptxgenjs 库直接生成 PowerPoint 文件：

```bash
# 安装依赖
npm install pptxgenjs

# 运行生成脚本
node generate-ppt.js
```

**优点**：
- 生成真正的.pptx 文件（137KB）
- 支持自定义样式（颜色、字体、布局）
- 支持演讲备注
- 支持表格、两列布局等复杂排版

**缺点**：
- 需要 Node.js 环境
- 样式需要手动配置代码

**适用**：快速生成带样式的 PPT 初稿

> 💡 **提示**：课程已提供生成脚本（`generate-ppt.js` 和 `pptx-generator.js`），可以直接使用或根据需要修改。

---

**方案 B：使用 VBA 宏自动转换（备选）**

让 OpenCode 生成 VBA 宏代码，自动将 Word 大纲转换为 PowerPoint 幻灯片：

```
请生成一个 VBA 宏代码，
可以将 PPT 演讲大纲.docx 的内容自动转换为 PowerPoint 幻灯片。
```

**优点**：一键生成，无需额外依赖  
**缺点**：样式简单（白纸黑字），需要手动美化  
**适用**：快速生成初稿，后续手动调整样式

> 💡 **提示**：生成 VBA 代码后，在 PowerPoint 中按 `Alt+F11` 打开 VBA 编辑器，粘贴代码运行即可。

---

**方案 C：手动复制（质量可控，推荐用于正式场合）**

1. 打开 PowerPoint，创建新演示文稿
2. 根据 Word 大纲，逐页创建幻灯片
3. 复制要点到幻灯片，演讲备注到备注区
4. 使用 PowerPoint 设计师工具美化样式

**优点**：样式精美，完全可控  
**缺点**：耗时较长  
**适用**：正式汇报、重要会议

## ❓ 常见问题

### Q1: 为什么生成了 `项目状态报告_v2.docx` 这样的文件？

**原因**：原文件已被打开或占用，OpenCode 无法覆盖。

**解决**：
1. 关闭已打开的 Word 文档
2. 让 OpenCode 重新生成，或手动将 `_v2` 版本重命名

**预防**：生成文件前先关闭目标文件。

---

### Q2: 图表中的中文显示为方框怎么办？

**原因**：matplotlib 默认字体不支持中文。

**解决**：输入以下提示词让 OpenCode 修复：

```
请在生成图表的 Python 脚本中，添加中文字体配置：

import matplotlib.pyplot as plt
plt.rcParams['font.sans-serif'] = ['Microsoft YaHei', 'SimHei']
plt.rcParams['axes.unicode_minus'] = False

然后重新运行脚本生成报告。
```

> 💡 **已优化**：本课程的提示词已包含中文字体配置要求，第一次生成就应该正常显示中文。

---

### Q3: 提示"Permission denied"或"Device or resource busy"怎么办？

**原因**：Word/Excel 文件已被打开，无法写入。

**解决**：
1. 关闭已打开的文件
2. 让 OpenCode 重新生成

---

### Q4: 生成的内容和格式不符合要求怎么办？

**这是正常的！** OpenCode 是根据你的提示词生成内容的，如果结果不满意：

1. **明确指出问题**：如"风险分析表格缺失"
2. **告诉它如何修改**：如"请添加一个表格，包含风险等级和缓解措施"
3. **迭代优化**：可能需要 2-3 轮对话才能达到理想效果

**示例**：

| 轮次 | 你的提示词 | 结果 |
|------|-----------|------|
| 第一次 | "请生成一份项目状态报告" | 只有文字，没有数据表格 |
| 第二次 | "请添加项目进度表格，包含完成百分比和预计日期" | 有表格，但数据不完整 |
| 第三次 | "请补充表格数据，包含各项目的具体进度和负责人" | 完美 ✅ |

> 💡 **鼓励**：不要担心需要多轮对话，这正是 AI 办公的优势——你可以像和同事交流一样不断调整，直到满意为止。AI 的输出偏差是 feature，不是 bug。

---

### Q5: OpenCode 生成了英文内容怎么办？

**原因**：提示词中没有明确指定语言。

**解决**：

```
请使用简体中文重新生成，专业术语保留英文原文（如 ROI、KPI、SLA）。
```

**预防**：在提示词开头明确"使用简体中文"。

---

### Q6: OpenCode 说"Could not find oldString"怎么办？

**原因**：代码编辑时找不到要修改的内容。

**解决**：让 OpenCode 重新读取文件后重试，或者直接生成新脚本。

---

### Q7: 如何生成带样式的 PPT 文件？

**推荐方案**：使用 Node.js + pptxgenjs（Anthropic 官方 pptx skill）

**步骤**：
1. 安装依赖：`npm install pptxgenjs`
2. 使用提供的生成脚本：`node generate-ppt.js`
3. 获得带样式的.pptx 文件（137KB）

**样式特点**：
- 深蓝色主题（`#1E2761`）
- 微软雅黑中文字体
- 支持标题页、内容页、两列布局、表格页
- 包含演讲备注

**进阶美化**：
1. 在 PowerPoint 中打开生成的文件
2. 使用"设计师"工具自动美化
3. 应用 PowerPoint 内置模板
4. 调整配色和布局

**预期管理**：当前方案生成基础样式的 PPT，正式场合建议进一步美化。
